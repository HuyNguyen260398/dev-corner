Dev Corner — Privacy Policy

Effective date: 2026-06-28

Summary

Dev Corner builds a reading digest from blog sources you explicitly save. Your subscriptions, extracted post metadata, favorites, settings, and crawl state are stored locally in your browser. Dev Corner has no account, backend, analytics, advertising, or telemetry, and the developer does not receive or read this data.

Data handled locally

  • Source URLs and titles you choose to subscribe to.
  • Post URLs, titles, summaries, publication times, and permitted thumbnail URLs extracted from those sources.
  • Favorite-post snapshots you explicitly create.
  • Daily crawl and notification preferences.
  • Crawl queue, timestamps, and source-specific error messages needed for reliable local operation.

This information is stored in extension IndexedDB and chrome.storage.local. It is not transmitted to the developer or a developer-controlled service.

Network requests

Dev Corner requests RSS, Atom, and HTML content only from source origins you explicitly save and grant through Chrome's per-origin permission prompt. It may also load an HTTPS thumbnail from the saved source host or one of that host's subdomains. Thumbnail URLs on sibling or unrelated domains are not requested by the extension.

The selected source operator or its thumbnail subdomain receives ordinary network information, such as your IP address, user agent, requested URL, and request time. Dev Corner does not add an account identifier or tracking identifier.

If you explicitly save and grant an HTTP-only source, that connection is not encrypted. Dev Corner never downgrades an HTTPS source to HTTP.

Permissions

  • activeTab — temporarily reads the current page URL and title after you invoke Dev Corner, so you can subscribe to that page. It does not monitor general browsing.
  • storage — persists local settings and resumable crawl state.
  • alarms — schedules the optional daily crawl and resumes bounded crawl work without persistent timers.
  • contextMenus — adds “Save to Dev Corner” for a page or link you explicitly select.
  • notifications — sends an optional completed-digest alert. Notifications are off by default.
  • optional host permissions — grants access only to source origins you explicitly subscribe to and approve.

Retention and deletion

Normal post metadata is retained for the latest seven crawl days. Unsubscribing from a source removes that source, its normal posts, and its origin grant when no other source uses the origin. Favorite posts are independent snapshots and remain until you remove them. Uninstalling Dev Corner removes its local extension storage.

Sharing and prohibited uses

Dev Corner does not sell, share, or transfer locally stored data to advertisers, data brokers, or other third parties. It does not use data for personalized advertising, lending, creditworthiness, or unrelated purposes, and no human is allowed to read it.

Dev Corner's use of information received from Chrome APIs adheres to the Chrome Web Store User Data Policy, including the Limited Use requirements.
